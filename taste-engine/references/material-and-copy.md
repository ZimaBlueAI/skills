# Material & copy — sourcing and wording with taste

The dials set the frame; material and copy fill it. Generic assets and AI-phrasing undo
good layout instantly. These are the rules that keep content honest and specific.

---

## Material（选材）

**Images / assets**
- Real images only. Default sources: Wikimedia Commons, The Met Open Access, Unsplash.
  Never lorem / never gray placeholder blocks in a delivered artifact.
- Match subject and era to the content. A stock "team high-five" is slop; a real product
  shot, a real chart, a real screenshot is material.
- Icons: geometric glyphs or hairline-stroke marks, one consistent family. **emoji are not
  icons** (they may appear inside a quoted original, not as UI furniture).

**Data**
- Use true values. Mark estimates `[估]` and missing values `[N/A]`. **Never fabricate a
  number** — one invented figure burns the whole deliverable's trust.
- Cite source lines (`来源：…`) for any non-obvious figure.
- Charts inherit the deliverable's palette; grids stay faint; legends hidden unless needed.

**Color as material**
- Pull accent colors from the subject when possible (brand, product, domain), not a random
  palette. Keep the semantic trio (risk / positive / focus) reserved.

---

## Copy（文案）

**Titles are conclusions, not labels**
- "营收四个季度翻倍" > "营收分析". "Windows 没有生产级 eBPF 等价物" > "Windows 平台现状".
- The reader should get the takeaway from the title alone; the body proves it.

**Kill these patterns**
- "不是 X 而是 Y" 句式 — overused, hollow.
- 黑话堆叠：赋能 / 抓手 / 闭环 / 痛点 / 打法 / 心智 / 颗粒度 — say the concrete thing instead.
- Filler adjectives: crucial / robust / comprehensive / seamless / 强大 / 全面 / 一站式.
- Hedging stacks: "可能也许大概会有助于" — commit or cut.

**Voice**
- Concrete over abstract: name the file, number, action, user-visible effect.
- Short paragraphs. One idea per line in lists.
- Competitor / external claims: use **their** words, don't reframe in your own jargon.
- Honest about uncertainty: `[估]` / `[N/A]` beat a confident wrong number.

**The read-aloud test (for decks)**
- If a line is meant to be spoken, write it the way you'd say it out loud, not the way you'd
  write a slide. Speaker-script lines should be sayable verbatim.

---

## Quick check

> Could a competitor publish this exact sentence about themselves? If yes, it's too generic
> — make it specific to this subject, with a real number or a named thing.
