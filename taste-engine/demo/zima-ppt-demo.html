两个软桥接

需要时接上，不装也不影响

huashu-design

Node + Playwright + ffmpeg。HTML → MP4 / 60fps / GIF、设备外壳、动画引擎、20 设计哲学、5 维专家评审。viz-deck 模式 2–4 与 viz-charts motion 模式调用。

Taste · 反 AI slop

更有口味，靠的是减法

配色

一页强调色 ≤ 2 种；语义色只给语义（红=风险/降，绿=正向），不当装饰。

排版

靠留白和发丝描边撑结构；大面积铺色是反模式；数字 tabular + 800 粗体。

选材

真图真数；估值标 [估]、缺失标 [N/A]；不用 lorem，不编造。

文案

用结论句不用名词（"营收四季翻倍"非"营收分析"）；不堆黑话，不写"不是 X 而是 Y"。

动效

只做入场不炫技；禁无限循环 / 霓虹辉光 / 粒子；带 reduced-motion 降级。

图标

emoji 不当图标用；用几何符号或发丝描边小图，统一调性。

专业 = 安静 · 标准取自 huashu-design critique-guide07 / 09

节奏 60s：六格各 8s 扫过，落在"动效"和"文案"两格。

话术 "有口味"不是加东西，是会做减法。一页两种强调色封顶、数字用真值、动效只做入场——克制本身就是高级。

Taste Engine · 三档可调

同一套内容，三个旋钮调出三种气质

DESIGN_VARIANCE

0 居中对称安全栅格
1 非对称黄金分割、错位标题
2 编辑式破格、对角流向、超大字

MOTION_INTENSITY

0 静止（reduced-motion）
1 入场揭示、数字滚动、条形填充
2 滚动视差、弹簧物理、3D 漂移

VISUAL_DENSITY

0 一屏一念、大留白
1 Bento 平衡
2 仪表盘密度、多面板、紧行距

每个技能有默认档：biz 报告 = 0/0/1（守零动效扫读）· zima 讲演 = 1/1/1 · viz 发布 = 2/2/1。

taste 是"加"出来的——各技能既有基调不被覆盖08 / 09

节奏 60s：三个旋钮各 14s，默认档 18s。

话术 三个旋钮：张力、动效、密度。同一份内容，biz 报告调成 0/0/1 守扫读，发布会调成 2/2/1 出片。风格是加出来的，不是覆盖出来的。
