AI NEBULA

用 Agent 改造周报：从收集信息到生成判断

Demo 主题：AI 星云风。这是一份可复制到公众号编辑器的预览样张。

一个好 Agent 不只是会写字，它应该知道什么时候查资料、什么时候停下来问人。

01 / AI NEBULA

场景：周五下午的重复劳动

每周五，项目经理要从研发看板、客服记录、销售反馈和数据后台里拼一份周报。真正有价值的是判断风险和确定下周动作，但时间大多花在复制信息上。

我们希望 Agent 接管信息收集和初稿生成，人只负责校正事实和判断优先级。

02 / AI NEBULA

Agent 的输入输出

输入包括四类：本周完成的任务、延期任务、客户反馈、关键指标变化。输出不是流水账，而是一份可以直接给团队看的结构化周报。

输入：Jira 任务、客服摘要、销售备注、指标截图

输出：进展摘要、风险列表、下周动作、需要负责人确认的问题

03 / AI NEBULA

能力边界

Agent 可以归纳重复反馈，也可以把风险按严重程度排序。但它不能替负责人承诺交付日期，也不能凭空补齐缺失数据。遇到缺失字段时，它必须标记为“待确认”。

待确认 比 自动脑补 更重要。因为周报的目标是让团队对齐现实，而不是制造一份看起来完整的文档。

04 / AI NEBULA

评测方式

我们用三项指标评估它是否值得上线：

• 信息遗漏率是否低于 5%。

• 风险排序是否被负责人认可。

• 初稿修改时间是否少于 15 分钟。

05 / AI NEBULA

下一步

先让 Agent 只负责一个项目组，连续跑 4 周。只要它能稳定减少整理时间，同时不增加事实错误，就可以进入第二阶段：自动生成跨项目周报。
