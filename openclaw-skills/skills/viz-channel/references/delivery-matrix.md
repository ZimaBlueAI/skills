# 形态决策矩阵 · 该生成什么、该怎么发

> 默认**动效 HTML**。只有明确信号才切换到 PPTX / MP4 / 单图。
> 这张表是"用户的话 → 生成什么 → 用哪个技能哪个模式 → 怎么发飞书"的全链路。
>
> **风格无关形态**:不管出 HTML / PPTX / MP4 / 图表,**视觉风格默认都走 ZimaBlue Editorial
> 暖纸编辑风**(见 `default-style.md`)。形态决定"发什么文件",风格决定"长什么样",两者独立。

## chart vs 3D KG vs deck —— 别混

- **单张/几张图表**:用户要的是"一张图说明数据" → `viz-charts` 直接出图,不套 deck 外壳。
- **3D 知识图谱**:用户要"实体关系/网络/图谱" → `viz-charts` 的 3D KG(WebGL,可旋转缩放)。
  必发 HTML(PNG 丢交互)。提醒下载浏览器打开。
- **整份报告/演示**:多页叙事 + 图表 → `viz-deck`,图表由 viz-deck 内部调 viz-charts 嵌入。
  → 这种情况**只发一个 deck 文件**,不要把图表再单独发一遍。

## HTML vs PNG(图表/图谱专属)

| 选 HTML 当… | 选 PNG 当… |
|---|---|
| 要交互(hover/旋转/筛选) | 只要"看一眼",贴群里直接显示 |
| 3D 知识图谱(必须 HTML) | 要嵌进别的文档/微信 |
| 数据会让用户想钻取 | 静态汇报截图 |

PNG 在飞书是**图片消息**,群里直接显示缩略图;HTML 是**文件消息**,要下载。
拿不准 → 默认 HTML(信息不丢),回执里说"要静态图我转 PNG 给你"。

## 形态升级路径(用户改主意时)

```
动效 HTML ──"要能改"──▶ 可编辑 PPTX(模式5)
动效 HTML ──"群里直接播"──▶ MP4(模式4)
动效 HTML ──"只要那张图"──▶ 抽出图表单发(viz-charts)
```
同一主题/数据复用,换模式重生成,覆盖式再发一次。

## 飞书侧硬约束

| 约束 | 数值 | 触发时怎么办 |
|---|---|---|
| 文件消息大小上限 | ~30MB | 超了:压缩 / MP4 降码率 / 转云文档链接 |
| MP4 建议规格 | 1920×1080, ≤8s 段, ≤30MB | viz-deck 模式4 默认就在范围内 |
| 上传文件/视频权限 | `im:resource` | 缺则报 91402,补权限重发版 |
| HTML 动效 | 飞书内不渲染 JS | **必须**提醒下载用浏览器打开 |

## 自检清单(交付前过一遍)

- [ ] 文件真生成了?(路径存在、大小 > 0 且合理)
- [ ] 形态对得上用户的话?(没把"要能改"做成了 HTML)
- [ ] HTML/PPTX/MP4 的附言提醒对不对?
- [ ] chat_id 是**当前会话**?(没发错人/错群)
- [ ] 大小没超 30MB?
