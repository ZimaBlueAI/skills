# 频道对话协议 · 把"聊一句"变成"出一份成品"

> 频道是聊天场景,不是表单。节奏:**确认 → 至多两问 → 动手 → 回执**。
> 全程默认值给足,用户不答就用默认推进,绝不卡住。

---

## 状态机

```
收到请求(@机器人 + 一句需求)
        │
        ▼
[确认+解析]  回一句"收到,我来做 <理解到的东西>"  ← 让用户知道你听懂了
        │
        ▼
[需求是否够清?] ──够清──▶ 直接进[生成]
        │ 不够清
        ▼
[至多两问]  主题/数据? + 形态?(给默认)   ← 一条消息问完
        │
        ▼
[生成]  触发 viz-deck / viz-charts,按形态走对应模式
        │
        ▼
[自检]  文件真生成了吗?路径在、大小合理?
        │
        ▼
[交付]  channel_send.sh 发回当前 chat
        │
        ▼
[回执]  "已发 <文件>,<怎么看>"  ← 发成功后才说
```

## 话术模板(可直接改用)

**① 确认 + 解析(收到就回,别让用户等)**
> 收到 ✦ 我来做一份「<主题>」的<形态>。先跟你确认两点 👇

**② 至多两问(一条消息内,带默认)**
> 1）内容/数据:你这边有要用的数据或要点吗?没有我先用占位结构,你看了再补。
> 2）形态:默认出**动效 HTML**(最轻、含图表和 3D 图谱,下载用浏览器看)。
>    如果你要**能在 PowerPoint 改的**就说一声,我出可编辑 PPTX;要**群里直接播**就出 MP4。

**③ 开工(用户答了或用默认)**
> 好,按动效 HTML 来,稍等,我做好直接发这个会话。

**④ 回执(文件发成功后)**
> 已发 ✦ 「<主题>」动效报告(deck.html)。**下载后用浏览器打开**就能看到完整动效、
> 图表和 3D 图谱。要改成可编辑 PPT 或加页,直接说。

## 决策捷径(什么时候可以不问直接做)

- 用户已经把**主题 + 形态**都说清了(如"做个能改的 Q2 业绩 PPT")→ 不用问,直接做可编辑 PPTX。
- 用户只说了主题、没说形态 → **默认动效 HTML**,做完在回执里告知"要别的形态可切换"。
- 用户给了明确数据/文档 → 直接用,不要再问"有没有数据"。
- 用户说"随便/你看着办" → 全默认:动效 HTML + viz-deck 设计方向顾问默认风格。

## 多轮迭代(用户在频道里说"改一下")

频道对话天然多轮。用户收到后常说"把第二页的图换成饼图""加一页结论""换个配色"。
- 在**同一产物**上改,重新生成,**再发一次**(覆盖式交付),回执说明改了什么。
- 不要每次都重新问需求——记住本会话已确定的主题/形态/风格。
- 会话上下文由平台维持(OpenClaw/Hermes 的 session 绑定到该 chat)。

## 群聊 vs 单聊

- **单聊**:`chat_id` 即该用户会话;直接 `--to-current`。
- **群聊**:必须被 @ 才响应;成品发回群(`oc_` 开头的 chat_id),群成员都能下载。
  涉及敏感内容时,先在群里问"发群里还是私发给你?",私发就用提问人的 open_id。

## 失败处理(频道里要给人话,不要甩堆栈)

- 凭证/权限错(91402 等)→ 回:"发送遇到权限问题(缺 im:resource),
  我已生成文件在 <路径>,管理员补权限后我重发。"
- 文件超 30MB → 回:"成品有点大(<N>MB,超飞书上限),我压一版 / 转成链接发你。"
- 生成失败 → 说清哪一步失败,不要假装成功。
