# gzh-channel · 草稿确认卡片协议 + 频道话术

## 1 · 草稿确认卡片(interactive)结构

`gzh_card_send.py --draft` 生成的飞书卡片(经典 v1 结构,`msg_type=interactive`):

```json
{
  "config": { "wide_screen_mode": true },
  "header": {
    "template": "green",
    "title": { "tag": "plain_text", "content": "📝 公众号排版 · Markdown 草稿确认" }
  },
  "elements": [
    { "tag": "markdown", "content": "**《文章标题》**\n章节 **5** · 图片 3 · 代码块 2 · 约 **3200** 字\n\n01 · 章节一\n02 · 章节二\n…" },
    { "tag": "hr" },
    { "tag": "markdown", "content": "````markdown\n# 文章标题\n\n> 开头引言…\n\n## 章节一\n…\n````" },
    { "tag": "hr" },
    { "tag": "note", "elements": [
      { "tag": "plain_text", "content": "确认无误请回复「确认」开始排版;要调整直接说改哪里(标题/章节/某段文字)。" }
    ] }
  ]
}
```

要点:

| 设计决策 | 原因 |
|---|---|
| 代码块用 **4 个反引号** 围栏(````` ````markdown `````) | 文章正文里常有 ``` 代码块,3 反引号会提前闭合 |
| 全文内嵌上限 6000 字符(`--max-inline`) | 飞书卡片体积有上限;超出改「摘要卡片 + 完整 .md 文件消息」 |
| 确认走**文字回复**,卡片不放按钮 | 按钮回调要网关接卡片回传事件(callback URL / 长连接订阅),文字回复零配置可用 |
| interactive 失败自动回退 text + 代码围栏 | 老租户 / 权限不全时流程不断,内容一样送达 |
| header 用 green 模板 | 与「草稿待确认」的轻量语气匹配;交付回执不用卡片,纯文本即可 |

## 2 · 长稿策略(>6000 字符)

1. 卡片:结构摘要(标题 / 章节列表 / 字数 / 图片数)+ 开头 500 字预览代码块 + 「全文见 .md 附件」。
2. 紧跟一条**文件消息**:完整 `draft-xxx.md`(飞书里点开可读,也可下载本地编辑)。
3. 用户改稿的两种方式都接:频道里说「第几章怎么改」(你来改),或用户自己改完 .md 回传(以回传为准)。

## 3 · 频道话术模板

**收稿回执(第 1 步后)**:
> 收到,整理成 Markdown 结构稿了,卡片在下面——确认结构没问题我就开排。

**草稿卡片后等确认(第 2 步)**:
> (卡片自带提示,不用重复啰嗦;用户 10 分钟没回可轻提醒一次,只提醒一次)

**选主题(第 3 步,一条消息问完)**:
> 结构确认 ✓。主题建议用**摸鱼绿**(教程/盘点类首选),直接排?
> 也可换:红白(观点力量感)/ 石墨极简(科技评论)/ 留白禅意 / 摸鱼票据 / 橄榄手记。

**交付附言(第 5 步,`--note` 固定要素)**:
> 公众号排版好了 ✦ 下载「`_预览.html`」用**浏览器打开** → 点右上角「**复制到公众号**」→
> 到公众号编辑器 Ctrl/⌘+V 粘贴即可,样式不会丢。另一个文件是干净正文,手动粘贴兜底用。

**回执(第 6 步)**:
> 已发 2 个文件:预览页(一键复制)+ 干净正文(兜底)。主题:摸鱼绿;
> 校验:ERROR×0。要换主题或改哪段,直接说。

## 4 · 排错速查

| 现象 | 原因 | 处理 |
|---|---|---|
| 卡片发送 code=230099 / 组件报错 | 租户卡片组件版本不支持 markdown 代码块 | 脚本已自动回退 text;无需处理 |
| code=91402 / permission denied | 缺 `im:resource`(上传文件) | 飞书开发者后台补权限重新发版 |
| `--to-current` 解析不到 | 网关没注入 chat 环境变量 | 从消息事件取 `chat_id`,`--to oc_xxx --to-type chat_id` |
| 用户说粘贴后样式丢了 | 大概率没用预览页复制按钮 / 用了干净正文直接开文件复制 | 引导用预览页按钮;或检查是否跳过了 validate |
| 卡片里代码块被截断 | 稿超 `--max-inline` 却手动强塞 | 走长稿策略,别调大上限硬塞 |
