02 Installation

5 分钟跑通最小可用

两种安装模式：单项目（验证用）或全局（所有项目共享）。建议先在单项目里跑通，再决定是否全局化。

Path A · 推荐

单项目安装

把三个 zip 解到目标项目根目录（与 .git 同级）。

git clone --depth=1 \
  https://github.com/ZimaBlueAI/skills.git
cd skills/claude-code-skills

# 解压到你的项目根
unzip -o biz-decision-stack/biz-decision-stack.zip \
      -d /path/to/your-project/
unzip -o viz-deck/viz-deck.zip \
      -d /path/to/your-project/
unzip -o viz-charts/viz-charts.zip \
      -d /path/to/your-project/

解压后会有 .claude/agents/ 与 .claude/skills/ 两个目录。

Path B

全局安装

装到 ~/.claude/，所有项目共享。

unzip -o biz-decision-stack/biz-decision-stack.zip \
      -d ~/
unzip -o viz-deck/viz-deck.zip -d ~/
unzip -o viz-charts/viz-charts.zip -d ~/

三个 skill 会落在 ~/.claude/skills/{biz-html-viz,viz-deck,viz-charts}/，agents 在 ~/.claude/agents/。

提示：全局装的优势是不用每个项目重复解压；劣势是版本升级要谨慎，会一次性影响所有项目。

验证安装

在 Claude Code 里运行：

> /skills

应能看到列表里有：

可选：装 huashu-design 启用 v2 高级模式

v2 的 prototype / slide-deck / motion-stage 三个模式需要 huashu-design 提供工具链。不装也能用——只是会降级到 v1 能力。

git clone --depth=1 https://github.com/alchaincyf/huashu-design.git \
  ~/.claude/skills/huashu-design
cd ~/.claude/skills/huashu-design

# huashu 自身不附带 package.json，用本仓库推荐清单
cat > package.json <<'JSON'
{
  "name": "huashu-design-runtime",
  "version": "1.0.0",
  "private": true,
  "dependencies": {
    "playwright": "^1.48.0",
    "sharp": "^0.33.5",
    "pptxgenjs": "^3.12.0",
    "pdf-lib": "^1.17.1"
  }
}
JSON

npm install
npx playwright install chromium

# ffmpeg 必须在 PATH
ffmpeg -version

注意

不要给 huashu 的 package.json 加 "type": "module"——会破坏它内部的 CommonJS 脚本（如 render-video.js）。脚本扩展名约定：.js = CJS，.mjs = ESM，混用即可。

可选：装 ppt-master 启用 v3 真正可编辑 PPTX

v3 新增。三个 skill 现在都可以把产出落地为每个元素都可点击的 PowerPoint 文件——不是图片嵌入，是原生 OOXML <p:sp> shapes。不装也能用——只是失去模式 5 / 决策 PPTX / 数据绑定 chart 这三条 v3 路径。

git clone --depth=1 https://github.com/hugohe3/ppt-master.git \
  ~/.claude/skills/ppt-master
cd ~/.claude/skills/ppt-master

# 隔离的 Python 3.10+ venv
python -m venv .venv

# Windows

  reportlab Pillow numpy
# macOS/Linux

# 验证

双桥接的关系

huashu-design（Node + Playwright + ffmpeg）管动画 / 视频 / 设计哲学 / 5 维评审；ppt-master（Python + python-pptx + svglib + edge-tts）管真正可编辑 PPTX / TTS 旁白 / 母版继承 / 数据绑定图表。两者并列，不互相替代。可以只装一个，或两个都装。

04 v2 Core — viz-deck Four Modes

四种产出形态，一套视觉语言

v2 把 viz-deck 从单模 keynote-report 扩展为 4 模产出，v3 再加上模式 5 · pptx-deck，共五种产出形态共享深空蓝紫青 + Inter 设计语言。Agent 按用户自然语言自动路由到合适模式。

① keynote-report MODE 01 · v1

讲演级长滚动报告，受众客户/投资人/全员 keynote。

  • 3 模板：stage-report · architecture-deep · competitive-landscape
  • 章节 5-8 章，每章 250-600 字
  • 含 reveal 滚动入场、prefers-reduced-motion 降级
  • 竞品模板含 web_search 三档自适应调研
触发： "讲演" · "阶段汇报" · "架构深研" · "竞品对比"

② prototype MODE 02 · v2

HTML 高保真原型，受众 PM / 设计评审 / 内部客户。

  • 1 模板：prototype-shell.html（iOS 15 Pro 默认）
  • AppPhone 状态管理：home → detail → settings 切屏
  • 设备外壳来自 huashu：ios_frame.jsx / android_frame.jsx / macos_window.jsx / browser_window.jsx
  • 真实素材：Wikimedia / Met / Unsplash
  • 交付前 Playwright 点击测试每个按钮
触发： "做原型" · "iOS 原型" · "hi-fi" · "可点击 demo"

③ slide-deck MODE 03 · v2

HTML 演讲幻灯片，可一键导出可编辑 PPTX。

  • 1 模板：slide-deck.html（键盘 ← → / 空格翻页）
  • 3-5 章节 / 12-25 总页 / 单页 ≤ 60 字
  • Speaker Notes 模式（按 s） / Overview 模式（按 o）
  • 真实文本框 PPTX 导出（不是图片嵌入）
  • 另支持横版 PDF / autoplay 录视频
触发： "做幻灯片" · "做 PPT" · "导出 PPTX" · "TED 风"

④ motion-stage MODE 04 · v2

HTML 动画一镜到底，导出 MP4 + GIF + BGM。

  • 1 模板：motion-stage.html（orbit 主视觉 + 三段动画时间线）
  • 录制要求：window.__ready = true 信号
  • 导出三件套：25fps + 60fps 插帧 + palette GIF
  • 6 首场景化 BGM（tech / educational / tutorial / ad）
  • 解说子模式：豆包 TTS + ducking 混音
触发： "导出 MP4" · "60fps" · "hero video" · "解说视频"

08 Troubleshooting

常见问题速查

十个真实遇到过的坑，每条都告诉你"为什么"和"怎么修"。

Claude 看不到 skill

检查 .claude/skills/ 是否在项目根（不是子目录）。每个 skill 必须有 SKILL.md 含 frontmatter（name: + description:）。最后确认 claude --version 是最新版。

render-video.js 报 "require is not defined"

huashu 目录的 package.json 不要加 "type": "module"——它会破坏 CommonJS 脚本。约定：.js = CJS，.mjs = ESM。

MP4 前 1-2 秒是黑屏

stage HTML 未设 window.__ready = true。huashu 用这个信号作录制 t=0 起点。模板已内置，自己写 stage 必须加：document.fonts.ready.then(() => { window.__ready = true; })。

MP4 录到了动画但太短

ECharts 动画在 setOption() 调用瞬间就开始播，可能在 __ready 之前已完成。修法：先创建空 chart 实例 → 等字体就绪 → 设 __ready → requestAnimationFrame 后再 setOption()。

Mermaid 离线渲染失败

第一次必须 npx playwright install chromium。沙箱环境下载失败会自动走 placeholder SVG 兜底，不会崩。终端看 [render-mermaid] 输出诊断哪一档失败。

3D KG 加载后空白

检查浏览器 WebGL 支持（chrome://gpu）。检查 {{KG_DATA_JSON}} 替换完整。节点 > 5000 会卡——拆图或换 Gephi 等专业工具。

PPTX 打开后文本变成图片

用错了脚本。html2pptx.js 适合任意 HTML 但偶尔降级到 PNG；export_deck_pptx.mjs 是 deck 专属，能识别 .slide / .eyebrow / .speaker-notes，保证文本是真实 PPT 文本框。

code-kg builder 跑出 0 边

builder 只解析仓库内相对 import。monorepo workspace 或 path alias（@/utils）可能识别不到。先 --group-by language 看节点；边的问题不影响节点。

doc-kg 抽出来概念太杂

调高 --min-mentions（默认 2，设 3-5 过滤低频）。调低 --top-terms（默认 40，设 20）。在 markdown 里多用 **显式加粗** 和 `code-term`，builder 优先抽这些。

字体看起来不对

检查能否访问 Google Fonts。断网会自动走系统兜底：JetBrains Mono → SF Mono → Menlo → Consolas → monospace（终端风）；Inter → ui-sans-serif → system-ui → PingFang SC（讲演风）。
