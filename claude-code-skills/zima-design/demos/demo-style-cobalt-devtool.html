WEBHOOK · LOCAL-FIRST

把线上 webhook
接进你的 localhost。

relay 捕获打到公网地址的每一个 webhook，落成本地文件，随时逐个重放到 localhost——断点调试第三方回调，不再部署十次才复现一次。

relay --help

RECORDED FIELDS · v0.9

01 · BODY

原始载荷

字节级保存请求体，含压缩编码与分块传输的还原。

02 · HEADERS

完整请求头

签名头（Stripe-Signature 等）原样保留，重放时签名校验照常通过。

03 · TIMING

到达时序

毫秒级时间戳与到达顺序，`--realtime` 按原始间隔重放。

04 · RETRY

重试链

同一事件的多次投递归并为一条链，重放时可只取末次。

05 · FILTER

事件过滤

`relay replay --only stripe.invoice.*` 通配符选择性重放。

06 · DIFF

响应对比

每次重放记录本地响应，`relay diff` 对比两次代码改动的行为差异。

03 · WATCH MODE

改代码，自动重放。

`relay watch` 盯住你的源码目录：每次保存，自动把选中的事件重放一遍，终端里直接看到断言结果。红了就改，绿了就提交。

  1. 1relay capture —— 录下这个下午的全部 webhook
  2. 2relay watch --only stripe.* —— 保存即重放
  3. 3断言写在 relay.assert.ts —— 状态码 / 落库行数 / 幂等性
  4. 4relay diff HEAD~1 —— 两个版本的行为差异一屏看完
