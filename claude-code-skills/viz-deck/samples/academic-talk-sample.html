Reflective loops cut LLM-generated deck error rate by 31%.

We show that a one-pass page-level critique loop catches more layout faults than a 5× larger model upgrade — at 1/40th the inference cost.

Speaker Bin Xiaofeng, Founder

Duration 25 min
ABSTRACT
We introduce a reflective-loop pass that re-scores every page of an LLM-generated slide deck on three objective dimensions (visual hierarchy, craft, functional fit) and emits per-page redo prompts for any page scoring below 7.0. Across 384 decks from three production skills, the loop reduced downstream human-edit rate from 47% to 16% (p < 0.001). We argue that this gain is not a model-size story: it comes from forcing the agent to look back, an architectural pattern adapted from PPTAgent (Wang et al., 2026).
01 CONTEXT · WHY DECKS REMAIN BAD

LLM-generated decks fail more on craft than on content.

Across the 384-deck sample, 73% of human edits targeted layout (overflow, spacing, hierarchy) — not factual rewrites. The bottleneck is the agent’s inability to look at its own output and notice that a single slide has 11 bullets.

  1. Layout errors dominate human edits. 73% of post-generation edits are layout-only (Wang et al., 2026, Table 4).[1]
  2. Larger models do not fix layout discipline. GPT-5-Opus and Claude Opus 4.7 produce 11+ bullet slides at similar rates (n=512, comparable to GPT-4 baseline).[2]
  3. Single-pass generation has no incentive to re-read. Decoder LLMs commit to a token sequence and cannot revisit; layout faults are tail-end artifacts of greedy decoding.[3]
02 HYPOTHESES

We test reflection against the obvious alternative — scaling.

Two competing hypotheses guide our intervention design: either layout quality is bound by raw model capability (Hypothesis A, scale matters), or it is bound by the absence of a self-review pass (Hypothesis B, reflection matters).

HYPOTHESIS A Model scale fixes layout.

  • Larger models hold more layout exemplars in-context.[4]
  • Predicted lift: edit rate drops 25-30% per 5× model size.
  • Cost implication: 40-50× inference cost per deck.

HYPOTHESIS B Self-review fixes layout.

  • A page-level pass surfaces what greedy decoding hides.[5]
  • Predicted lift: edit rate drops 30%+ regardless of base model.
  • Cost implication: 1.1-1.3× inference cost per deck.
03 METHOD · RESULTS

Reflection beats scale on both edit rate and cost.

We ran a 2×2 design (small vs large base model; with vs without reflection loop) on 384 decks across three skills (viz-deck, biz-html-viz, viz-charts). Edit rate was measured by counting unique human modifications in the 72 hours post-generation.

04 IMPLICATIONS

Look-back loops are a cheaper lever than scaling.

For organisations building production agentic pipelines on a budget, the result is direct: prefer a reflection pass before paying for a model upgrade.

  1. Reflection beats scale by 31 percentage points at 1/40th the cost. 384-deck factorial; effect persists across all three skills tested.
  2. The mechanism is forcing a second look, not generating new content. Loop touches only 6.3% of pages on average; remaining 93.7% pass on first generation.
  3. Will this generalize to longer artifacts (multi-document reports, codebases)? Next: replicate on multi-document research synthesis (Q3 2026, in progress).
Q&A ANTICIPATED QUESTIONS

Three questions I expect you will ask.

If your question is not here, raise it — I'll address it now or follow up via email.

How did you decide an edit was “layout” vs “content”?

We coded edits by their AST diff: changes to layout class, bullet count, or grid-column count classified as layout; changes to text content as content.[6]

Why did you not test the obvious cheap baseline — a static linter?

We did, in a follow-up. A handwritten layout linter caught 41% of the issues the LLM reflection pass caught, but missed all hierarchy-level faults (e.g., “this cover slide buries the verdict in slide 7”). Reflection is a partial superset of linting.

What are the limitations?

Three. First, our edit-rate metric proxies for quality, but doesn’t directly measure stakeholder satisfaction. Second, our 384 decks were all enterprise B2B — we have no data on consumer or creative-industry decks. Third, the reflection loop’s D1 (philosophy) and D5 (originality) dimensions are not objectively scoreable from the JSON spec; we currently default them to neutral 7.0.
